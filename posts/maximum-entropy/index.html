The principle of maximum entropy

2020-09-26

Say that you are a statistician and are asked to come up with a probability distribution for the current state of knowledge on some particular topic you know little about. (This, in Bayesian statistics, is known as choosing a suitable prior.) To do this, the safest bet is coming up with the least informative distribution via the principle of maximum entropy.

This principle is clearly explained by Jaynes (1968): consider a die which has been tossed a very large number of times NNN. We expect the average to be 3.53.53.5, that is, we expect the following distribution where Pn=16P_n = \frac{1}{6}Pn​=61​ for each nnn.

using AlgebraOfGraphics
using CairoMakie
using DataFrames

function plot_distribution(probabilities::Array)
  np = mapping([1:6] => :n, [probabilities] => :P_n) * visual(BarPlot)
  axis = (; xticks=1:6, limits=(nothing, (0, 1)), height=200)
  draw(np; axis)
end
plot_distribution([1/6, 1/6, 1/6, 1/6, 1/6, 1/6])

Instead, we are told that the average is 4.54.54.5. How likely is it for each number n=1,2,…,6n = 1,2, \ldots, 6n=1,2,…,6 to come up for the next toss?

Since we know that PPP always sums to 1, we have

∑n=16Pn=1. \sum_{n=1}^6 P_n = 1. n=1∑6​Pn​=1.

We also know that the average is 4.54.54.5, that is,

∑n=16n⋅Pn=4.5. \sum_{n=1}^6 n \cdot P_n = 4.5. n=1∑6​n⋅Pn​=4.5.

We could satisfy these constraints by choosing P4=P5=12P_4 = P_5 = \frac{1}{2}P4​=P5​=21​.

plot_distribution([0, 0, 0, 0.5, 0.5, 0])

This is unlikely to be the distribution for our data since it can be derived in relatively few ways, namely: by throwing only 444 and 555, and in such a way that the throws average to 4.54.54.5. A more likely distribution would be 

plot_distribution([0, 0, 1/4, 1/4, 1/4, 1/4])

This is still not the least informative distribution since it assumes n=1n = 1n=1 and n=2n = 2n=2 to be impossible events. Jaynes presents the straight line solution Pn=(12n−7)/210P_n = (12n - 7)/210Pn​=(12n−7)/210,

fg = plot_distribution([(12n - 7)/210 for n in 1:6])
fg

This solution would also fail if the mean would have been higher, because then P0=0P_0 = 0P0​=0 would occur again. The correct measure is the following information measure (Shannon, 1948), which is also known as information entropy,

SI=−∑ipilog⁡pi. S_I = - \sum_i p_i \log p_i. SI​=−i∑​pi​logpi​.

We can find pip_ipi​ for pi=1,2,…,6p_i = 1, 2, \ldots, 6pi​=1,2,…,6 by maximizing SIS_ISI​ for given constraints. This problem, known as MaxEnt, is hard to solve manually since there are 666 unknowns and various constraints. The solution can be approximated by rewriting it to a linear program.

Alternatively, analytic solutions exist for some subsets of this Shanon entropy maximization problem (Zabarankin & Uryasev, 2014). Here, we have that the mean is known (and nothing else), so the number of moments mmm is 111. Then, the maximum entropy distribution takes the form (Zabarankin & Uryasev, 2014; Eq. 5.1.7)

Pn=eρn∑n=16eρn,  for n=1,2,...,6. P_n = \frac{e^{\rho n}}{\sum_{n=1}^6 e^{\rho n}}, \: \text{ for } n = 1, 2, ..., 6. Pn​=∑n=16​eρneρn​, for n=1,2,...,6.

This function satisfies ∑n=16Pn=1\sum_{n=1}^6 P_n = 1∑n=16​Pn​=1 for any ρ\rhoρ. Now, we only have to find the ρ\rhoρ for which the average is 4.54.54.5. After some trial and error, you'll find that ρ=0.3715\rho = 0.3715ρ=0.3715 gives ∑n=16n⋅Pn≈4.501\sum_{n=1}^6 n \cdot P_n \approx 4.501∑n=16​n⋅Pn​≈4.501.

plot_distribution([0.0543, 0.0787, 0.114, 0.165, 0.240, 0.348])

This is the least informative distribution which satisfies the constraints. In other words, this is the distribution which can be obtained in the largest number of ways, given the constraints. For another example of maximum entropy distributions, see Chapter 10.1 of the book by McElreath (2020).
